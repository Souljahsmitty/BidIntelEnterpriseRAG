# Proposal Operations Acceptance Criteria

The feature is accepted only when code, tests, PDF, and video prove the same workflow.

## Acceptance Test

1. Extract RFP requirements.
2. Persist requirements in the compliance matrix.
3. Build a requirement trace from RAG evidence.
4. Create proposal workspace sections.
5. Insert approved content with evidence references.
6. Create and resolve a review issue while preserving history.
7. Recalculate the proposal health dashboard.
8. Confirm audit events exist.

## Required Proof

- Backend route proof.
- Frontend click proof.
- Test output.
- API smoke output.
- Failure/fix note.
- Build Bible/PDF update.
- Checkpoint MP4 for scoring.
